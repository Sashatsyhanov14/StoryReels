# План переработки UI (page.tsx) под текстовые чат-истории

Так как мы полностью отказались от генерации картинок и видео, старый код на 2200 строк с плеером Remotion и громоздкими компонентами больше не нужен. Мы заменим его на минималистичный и быстрый интерфейс в стиле мессенджера (Hooked / Telegram).

## Этап 1: Очистка проекта от старого кода
1. **Удалить зависимость от Remotion**:
   - Убрать импорты `RemotionPlayer`, `SceneElement` и связанные с ними стейты (progress, fps, frames).
   - Удалить файл `src/components/EpisodePlayer.tsx` (он больше не нужен).
2. **Очистить `page.tsx`**:
   - Удалить все стейты, связанные с `scenes`, генерацией видео и заглушками симуляции GPU.
   - Оставить только логику авторизации (Supabase/Magic Link) и вызовы API для получения/создания списка эпизодов.

## Этап 2: Верстка интерфейса мессенджера (Dark Mode)
1. **Основной экран (Чат)**:
   - Мобильно-ориентированный дизайн (mobile-first) с темной темой (`bg-zinc-950`).
   - Отображение пузырей сообщений (Chat Bubbles): 
     - Сообщения от `"Ты"` прижимаются вправо (зеленоватый/фиолетовый акцент).
     - Сообщения от других отправителей прижимаются влево (серый фон).
2. **Навигация и управление**:
   - Боковое меню (Sidebar) для списка прошлых историй.
   - Поле ввода для создания новой истории (Prompt input).

## Этап 3: Механика Tap-to-Reveal (Тап для чтения)
1. **Стейт `revealedIndex`**:
   - Пользователь видит только сообщения от `0` до `revealedIndex`.
   - При клике/тапе в любую свободную область экрана индекс увеличивается на 1.
2. **Имитация печати (Typing Indicator)**:
   - При переходе к следующему сообщению считываем его `typingDelayMs`.
   - Если задержка > 0, блокируем тап, показываем анимацию печати (`...` прыгающие точки) внизу экрана на заданное время, затем показываем сообщение.

## Этап 4: Интеграция Cliffhanger и Paywall
1. **Проверка блокировки**:
   - Если `revealedIndex` достигает значения `unlockedTillIndex` (полученного из БД) И у текущего сообщения стоит флаг `isCliffhanger: true`, дальнейшие тапы блокируются.
2. **Экран оплаты (Paywall)**:
   - При достижении клиффхэнгера чат размывается (backdrop-blur), поверх появляется красивое модальное окно: *"История прервалась на самом интересном! Узнай, что было дальше."*
   - Кнопка **"Разблокировать за 100 ₽"** отправляет запрос на `POST /api/yookassa/create` и редиректит на оплату ЮKassa.

## Этап 5: Создание новой истории (Генерация)
1. **Лоадер (Скелетон)**:
   - При отправке нового промпта через поле ввода, показываем стилизованный экран "ИИ пишет историю..." (анимация печатающегося текста).
   - Как только `POST /api/episodes/create` возвращает массив сообщений, сразу перекидываем пользователя в новый чат, где `revealedIndex = 0`.

---
*Ожидаемый результат: страница `page.tsx` сократится с 2200 до ~500-600 строк кода, станет работать моментально и будет выглядеть как нативный мессенджер.*
